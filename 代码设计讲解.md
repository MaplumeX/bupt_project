## 0. 先建立一张“全景图”

这个项目做的不是整颗 CPU，而是 **CPU 的“大脑”——控制器**。

TEC-8 实验台上已经有：

- 通用寄存器 R0~R3
- 算术逻辑单元 ALU（芯片 74LS181）
- 程序计数器 PC、地址寄存器 AR、指令寄存器 IR
- 存储器、数据开关、面板指示灯……

**缺的那一块就是“告诉这些部件在每个时刻该做什么”的控制器。**  
本仓库的 Verilog 模块，就是这块控制器的 FPGA 实现。

可以把它想成：

```text
面板开关 / 指令 / 标志位  ──►  控制器  ──►  一堆控制信号（写谁、算什么、走哪条总线）
                              （本代码）
```

## 1. 零基础概念速通

### 1.1 什么是 Verilog？

Verilog 是描述数字电路的语言。本文件里你会看到两类写法：

| 写法 | 含义 | 本代码中的例子 |
|------|------|----------------|
| `assign y = a & b;` | **组合逻辑**：输入变，输出立刻跟着变（像一堆门电路） | 绝大多数控制信号 |
| `always @(posedge clk)` | **时序逻辑**：只在时钟边沿才更新（像触发器/寄存器） | `Q`、`ST0`、`PCINC`、`LIR` |

> 类比：组合逻辑像“开关一拨灯就亮”；时序逻辑像“只有下课铃响才换教室”。

### 1.2 TEC-8 的时间节拍

TEC-8 用 **不定长机器周期**，常见两个节拍：

```text
W1 → W2 → W1 → W2 → ...
```

每个 W 节拍内部再分：

```text
T1 → T2 → T3
```

- **T1 / T2**：数据通路运算、总线传输的稳定期  
- **T3 上升沿**：真正把数据写进寄存器 / 存储器 / 标志位  
- **T3 下降沿**：控制器内部状态（模式 Q、阶段 ST0）切换

单步模式下，**每按一次 QD，只走完一个 W 节拍**（里面含 T1~T3）。

### 1.3 流水线 vs 顺序式（本项目为什么叫 pipelined）

| | 顺序式控制器 | 本项目（流水线） |
|--|-------------|------------------|
| 取指 | 单独占一拍（先取指，再执行） | **取指叠在执行节拍末尾** |
| 周期 | 每条指令多一拍“纯取指” | 短指令通常只需 1 拍（W1） |

好处：更快。代价：`LIR`（装指令）和 `PCINC`（PC+1）必须严格同步，否则会“吞指令”（详见第 7 节）。

### 1.4 硬布线控制器是什么？

不用微程序 ROM，而是用 **组合逻辑直接把“模式 + 指令 + 节拍”译成控制信号**。  
流程图画成什么，Verilog 的 `assign` 就写成什么——本代码基本是流程图的“布尔函数版”。

## 2. 模块接口：控制器的“插头”

```verilog
module pipelined_hardwired_controller (
    // ===== 输入 =====
    input wire [2:0] SWC_SWB_SWA,  // 面板模式开关
    input wire [3:0] IR7_IR4,      // 当前指令的高 4 位（操作码）
    input wire CLR,                // 复位，低有效
    input wire T3,                 // 时序节拍 T3
    input wire W1, W2,             // 机器节拍
    input wire C, Z,               // 进位 / 零标志

    // ===== 输出（控制数据通路）=====
    output SELCTL, ABUS, SBUS, MBUS,
    output M, CIN, DRW, LDZ, LDC, MEMW,
    output ARINC, PCADD, LPC, LAR, STOP, SHORT,
    output reg PCINC, LIR,         // 这两个是寄存器输出（时序化）
    output [3:0] S, SEL
);
```

### 2.1 输入在说什么？

| 信号 | 白话 |
|------|------|
| `SWC_SWB_SWA` | 面板三位开关，决定“现在是手动写寄存器 / 读存储器 / 自动取指执行……” |
| `IR7_IR4` | 指令操作码（ADD？LD？JMP？） |
| `W1/W2` | 当前处在第几拍 |
| `C/Z` | 上一条运算留下的标志，给 JC/JZ 用 |
| `T3` | 同步时钟（状态更新、取指锁存都靠它） |
| `CLR` | 复位 |

### 2.2 输出在说什么？（按功能分组）

**总线选择（同一时刻通常只开一条“数据来源”）**

| 信号 | 含义 |
|------|------|
| `SBUS` | 面板数据开关 → 总线 |
| `ABUS` | ALU 结果 → 总线 |
| `MBUS` | 存储器读出 → 总线 |

**ALU 控制（对接 74LS181）**

| 信号 | 含义 |
|------|------|
| `S[3:0]` | 运算功能选择 |
| `M` | 1=逻辑运算，0=算术运算 |
| `CIN` | 进位输入 |

**写使能 / 装载**

| 信号 | 含义 |
|------|------|
| `DRW` | 写通用寄存器 R0~R3 |
| `LPC` / `LAR` / `LIR` | 分别装载 PC / AR / IR |
| `PCINC` / `ARINC` | PC+1 / AR+1 |
| `PCADD` | PC 相对跳转（JC/JZ 用） |
| `LDZ` / `LDC` | 更新零标志 / 进位标志 |
| `MEMW` | 写存储器 |

**节拍与面板**

| 信号 | 含义 |
|------|------|
| `SHORT` | 本周期只要 W1，不要 W2（短指令） |
| `STOP` | 停机 / 单步暂停 |
| `SEL` / `SELCTL` | 手动读写寄存器时的寄存器号选择 |

## 3. 整体架构：代码的三层结构

源码可以按职责拆成三层：

```text
┌─────────────────────────────────────────────┐
│ ① 模式锁存 Q + 阶段位 ST0（时序）            │
│    回答：现在是什么工作模式？走到前半段还是后半段？ │
├─────────────────────────────────────────────┤
│ ② 取指时序化 PCINC / LIR（时序）             │
│    回答：这一拍要不要取下一条指令？            │
├─────────────────────────────────────────────┤
│ ③ 控制信号译码（组合 assign）                 │
│    回答：具体拉高哪些信号？                    │
└─────────────────────────────────────────────┘
```

对应源码大致分区：

1. `always @(negedge T3)` 锁存 `Q`，译出 `WRITE_REG / READ_REG / INS_FETCH / ...`
2. `always @(negedge T3)` 更新 `ST0`，`assign SST0 = ...`
3. `assign FETCH = ...` + `always @(posedge T3)` 驱动 `PCINC/LIR`
4. 一长串 `assign` 把指令/节拍变成具体控制信号

## 4. 第一层：工作模式与 ST0

控制器内部真正“记事”的状态其实很少，核心就三个名字：

| 名字 | 类型 | 一句话职责 |
|------|------|------------|
| **Q** | 3 bit 寄存器 | 记住“当前工作模式是哪一种” |
| **ST0** | 1 bit 寄存器 | 记住“这个模式里走到前半段还是后半段” |
| **SST0** | 1 bit 组合线 | “现在该不该把 ST0 从 0 置成 1”的条件 |

可以记成：

```text
Q    回答：现在在哪种工作模式？
ST0  回答：流程走到前半段还是后半段？
SST0 回答：这一拍结束时，要不要把 ST0 置 1？
```

三者关系：

```text
面板开关 SWC_SWB_SWA
        │  T3 下降沿采样
        ▼
       [Q]  ──► 译出 WRITE_REG / READ_REG / INS_FETCH / ...
        │
        │  再和 ST0、W1/W2 一起
        ▼
     控制信号译码（DRW、LAR、LIR ...）

       SST0 ──(T3 下降沿)──► [ST0] 从 0 变 1
```

---

### 4.1 Q 的作用（模式锁存）

#### 4.1.1 Q 是什么？

`Q` 是控制器内部的一个 **3 位寄存器**，用来保存当前面板工作模式的稳定副本。

面板上有三位模式开关 `SWC SWB SWA`，它们**不是**直接参与后面所有译码的；代码先在时钟沿把它们锁进 `Q`，之后所有“现在是什么模式”的判断都看 `Q`。

```verilog
reg [2:0] Q;

always @(negedge T3 or negedge CLR) begin
    if (!CLR) begin
        Q <= 3'b000;          // 复位：默认回到取指模式编码
    end
    else begin
        case (SWC_SWB_SWA)
            3'b100: Q <= 3'b100;  // 写寄存器
            3'b011: Q <= 3'b011;  // 读寄存器
            3'b000: Q <= 3'b000;  // 取指
            3'b010: Q <= 3'b010;  // 读存储器
            3'b001: Q <= 3'b001;  // 写存储器
            default: Q <= 3'b111; // 非法组合：不命中任何模式
        endcase
    end
end
```

#### 4.1.2 Q 译出的五种工作模式

| `Q`（= 开关编码） | 模式 | 代码里的信号 | 你在面板上要干什么 |
|-------------------|------|--------------|--------------------|
| 000 | 取指执行 | `INS_FETCH` | 装 PC，然后跑程序 |
| 001 | 写存储器 | `WRITE_MEM` | 往内存里写程序/数据 |
| 010 | 读存储器 | `READ_MEM` | 读内存内容核对 |
| 011 | 读寄存器 | `READ_REG` | 看 R0~R3 的值 |
| 100 | 写寄存器 | `WRITE_REG` | 手动给 R0~R3 赋初值 |
| 111 | （非法） | 全 0 | 不进任何分支，保护态 |

```verilog
assign WRITE_REG = (Q == 3'b100);
assign READ_REG  = (Q == 3'b011);
assign INS_FETCH = (Q == 3'b000);
assign READ_MEM  = (Q == 3'b010);
assign WRITE_MEM = (Q == 3'b001);
```

这五个信号是**互斥**的：同一时刻最多一个为 1。后面所有控制信号的 `assign`，几乎都先靠它们分大类。

#### 4.1.3 为什么一定要有 Q？不能直接用开关吗？

如果写成 `assign WRITE_REG = (SWC_SWB_SWA == 3'b100)`，会有两个问题：

**问题 A：机械开关会抖动**

拨开关时，中间可能短暂出现毛刺或非法组合。若组合逻辑直接拿开关值去驱动 `DRW/LPC/MEMW` 等，可能在抖动瞬间误写寄存器或存储器。

**问题 B：模式切换与本拍执行会“错位”**

`Q` 在 **T3 下降沿** 才更新，含义是：

```text
按一次 QD（一个 W 节拍）期间：
  全程使用“旧的 Q”决定控制信号
  本拍末尾 T3 下降沿，才把“新开关值”写进 Q
  → 新模式从下一拍才真正生效
```

因此：

> **拨完模式开关后，必须先单独按一次单步，把新模式锁进 Q；再拨数据/地址并操作。**

若把“拨模式”和“拨数据”挤在同一次单步里，会发生：

| 时刻 | 用的模式 | 实际效果（举例：想进写存储器） |
|------|----------|--------------------------------|
| 本次单步执行中 | **旧 Q**（比如还是取指 000） | 可能把开关数据错误地装进 **PC**（LPC），而不是 AR |
| 本次 T3 下降沿 | — | Q 才变成 001 |
| 下一次单步 | 新 Q=001 | 这时 SD 往往已被拨成下一个字节 → AR 装错 |

这就是 `测试操作注意事项.md` 陷阱 #1 的根因。

#### 4.1.4 Q 的时间行为（小白向时间轴）

```text
开关拨到 100（写寄存器）
        │
        │  此时 Q 可能仍是旧值（例如 000）
        ▼
按 QD ──► 本拍仍按旧 Q 跑完
        │
        └─ T3 下降沿：Q <= 100
        │
        ▼
再按 QD ──► 从这一拍起，WRITE_REG=1，真正进入写寄存器流程
```

**复位时**：`CLR=0` 把 `Q` 清成 `000`。所以松复位后默认偏向取指模式编码；但若你开关拨在别处，仍需按一次单步把实际开关锁进 Q。

#### 4.1.5 Q 不做什么？

- **不存数据**：Q 里不是运算结果，也不是地址。
- **不直接驱动面板灯**（本设计里 Q 也没引出端口）。
- **不负责“第几拍”**：那是 W1/W2 和 ST0 的事。

**Q 只回答一个问题：大方向上，控制器现在该走哪条总分支？**

### 4.2 ST0 的作用（阶段标志）

#### 4.2.1 ST0 是什么？

`ST0` 是控制器内部的 **1 位寄存器**，表示当前模式内部走到了哪个阶段：

| ST0 | 含义（统一说法） |
|-----|------------------|
| **0** | **准备/装载阶段**：先装起始地址，或先写前一半寄存器 |
| **1** | **执行/后续阶段**：连续读写，或正式译码跑指令 |

```verilog
reg ST0;   // 0 = 前阶段，1 = 后阶段
```

它和 `Q` 搭配：

```text
Q   决定“进哪扇大门”
ST0 决定“进门后先办手续，还是已经开始干活”
```

#### 4.2.2 各模式下 ST0 的具体含义

同一位 ST0，在不同模式里“前后阶段”内容不同，但结构类似：

| 模式 | ST0 = 0（前阶段） | ST0 = 1（后阶段） |
|------|-------------------|-------------------|
| **写寄存器 100** | 写 R0（W1）、写 R1（W2） | 写 R2（W1）、写 R3（W2） |
| **读寄存器 011** | （本设计读寄存器不依赖 ST0 分阶段，靠 W1/W2 选 R0R1 / R2R3） | 同左 |
| **写存储器 001** | 开关数据 → 装入 AR（起始地址） | 开关数据写入内存，AR+1 |
| **读存储器 010** | 开关数据 → 装入 AR（起始地址） | 内存读出显示，AR+1 |
| **取指 000** | 开关数据 → 装入 PC（程序入口），**不取指** | 按 IR 译码执行指令，并流水取下一条 |

**写寄存器完整 4 拍示意：**

```text
CLR 后 ST0=0
  QD1  W1 ST0=0  → 写 R0
  QD2  W2 ST0=0  → 写 R1，本拍末 ST0 置 1
  QD3  W1 ST0=1  → 写 R2
  QD4  W2 ST0=1  → 写 R3，本拍末 ST0 清 0（可继续再写一轮）
```

**写存储器示意：**

```text
CLR 后 ST0=0
  先锁存模式 Q=001
  QD  ST0=0 W1 → LAR（AR←起始地址），末尾 ST0→1
  QD  ST0=1 W1 → MEMW + ARINC（写一个字节，地址+1）
  QD  ST0=1 W1 → 再写下一字节……
```

**取指示意：**

```text
CLR 后 ST0=0，IR 被复位为 00H（NOP）
  先锁存模式 Q=000
  QD  ST0=0 W1 → LPC（PC←程序入口）+ SST0，本拍不取指
  之后 ST0=1：
    先按 IR=NOP 走一遍 → 发出 LIR/PCINC，真正取到程序第 1 条
    再进入正常执行循环
```

#### 4.2.3 ST0 在代码里如何被使用？

几乎所有“前后阶段动作不同”的信号都会带上 `ST0` 或 `~ST0`：

```verilog
// 启动时装 PC：取指模式 + 前阶段 + W1
assign LPC = (INS_FETCH & ~ST0 & W1) | (JMP & W1);

// 装 AR：读/写存储器的前阶段，或 LD/ST 的地址拍
assign LAR = ((READ_MEM|WRITE_MEM) & W1 & ~ST0) | ((LD|ST) & W1);

// 连续访问时 AR+1：读/写存储器的后阶段
assign ARINC = (READ_MEM|WRITE_MEM) & W1 & ST0;

// 写内存：写存储器后阶段，或 ST 指令的 W2
assign MEMW = (WRITE_MEM & W1 & ST0) | (ST & W2);

// 指令译码本身也要求已经进入后阶段
wire ADD = (IR7_IR4 == 4'b0001) & INS_FETCH & ST0;
```

关键点：

> **指令译码全部乘了 `INS_FETCH & ST0`。**  
> 所以 ST0=0 的启动拍即使 IR 里有值，也不会误执行 ADD/LD 等指令，只会做“装 PC”。

#### 4.2.4 ST0 怎么更新？（状态转移）

```verilog
always @(negedge T3 or negedge CLR) begin
    if (CLR == 0) begin
        ST0 <= 1'b0;                          // 复位：强制回前阶段
    end
    else if (SST0) begin
        ST0 <= 1'b1;                          // 置位：进入后阶段
    end
    else if (ST0 && W2 && WRITE_REG) begin
        ST0 <= 1'b0;                          // 仅写寄存器：后阶段 W2 结束后翻回 0
    end
end
```

用状态图表示：

```text
                 CLR
                  │
                  ▼
              ┌──────┐
       ┌─────►│ ST0=0 │◄──── 写寄存器 & ST0=1 & W2（T3↓）
       │      └──┬───┘
       │         │ SST0 有效（T3↓）
       │         ▼
       │      ┌──────┐
       └──────┤ ST0=1 │  取指/读/写存储器：通常保持为 1，直到再 CLR
              └──────┘
```

注意源码注释里的设计取舍：

- **写寄存器**需要 ST0 在 0↔1 之间循环（写完 R0~R3 还能再写一轮），所以有“W2 结束清 0”。
- **取指 / 读存储器 / 写存储器**一旦 SST0 把 ST0 置 1，一般就保持为 1，靠 CLR 才能回到 0。  
  上板换任务前按 CLR，就是为了让 ST0（以及 Q）回到干净初态。

#### 4.2.5 ST0 不做什么？

- **不是节拍信号**：W1/W2 才是“第几拍”；ST0 是“第几大阶段”。
- **面板上通常没有 ST0 灯**（它是模块内部 reg，未引出端口）。要用 `LAR/MEMW/ARINC/SEL` 等可见灯反推当前阶段，见 `测试操作注意事项.md` 陷阱 #2。
- **不等于“是否在跑程序”**：跑程序是 `INS_FETCH && ST0==1`；ST0=1 在写存储器模式下只表示“已经装过起始地址”。

### 4.3 SST0 的作用（ST0 的置位条件）

#### 4.3.1 SST0 是什么？

`SST0` 名字可理解为 **Set ST0（置位 ST0）**。  
它是一根 **组合逻辑线**（`wire`），不是寄存器：输入条件满足时立刻为 1，不满足立刻为 0。

真正把 `ST0` 改成 1 的动作，发生在 **T3 下降沿** 的 `always` 里采样到 `SST0==1` 时。

```text
SST0 = 1  只表示：“这一拍结束时，请把 ST0 置 1”
ST0  在 T3↓ 才真正从 0 变成 1
```

#### 4.3.2 代码定义

```verilog
assign SST0 = (ST0 == 1'b0) && (
           (Q == 3'b100 && W2) || // 写寄存器：前阶段的 W2 结束 → 进入写 R2/R3
           (Q == 3'b010 && W1) || // 读存储器：装完 AR 的那一拍
           (Q == 3'b001 && W1) || // 写存储器：装完 AR 的那一拍
           (Q == 3'b000 && W1)    // 取指：装完 PC 的启动拍
       );
```

拆成两层读：

1. **总开关**：`(ST0 == 0)`  
   已经是 1 就不必再置位，避免重复/乱置。
2. **分模式触发拍**：不同模式在不同节拍“完成前阶段工作”，才允许进入后阶段。

| 模式 | 何时 SST0=1 | 前阶段刚做完的事 |
|------|-------------|------------------|
| 写寄存器 | `ST0=0` 且 **W2** | 写完 R1，准备进入写 R2/R3 |
| 读存储器 | `ST0=0` 且 **W1** | 已把起始地址装进 AR |
| 写存储器 | `ST0=0` 且 **W1** | 已把起始地址装进 AR |
| 取指 | `ST0=0` 且 **W1** | 已把程序入口装进 PC |
| 读寄存器 | （表达式里没有） | 不靠 ST0 分前后阶段 |

为什么写寄存器要用 **W2** 才 SST0，而存储器/取指用 **W1**？

- 写寄存器前阶段要写 **两个** 寄存器：R0 在 W1、R1 在 W2，必须等 W2 做完才翻阶段。
- 读/写存储器、取指的前阶段都是 **单拍 SHORT**：一个 W1 装完 AR 或 PC，当拍就可以置位。

#### 4.3.3 为什么 SST0 必须看 Q，不能看原始开关？

源码注释写得很清楚：若 SST0 误写成看 `SWC_SWB_SWA`，拨开关瞬间可能出现：

```text
当前真正执行的模式仍由旧 Q 决定
但 SST0 已按新开关算出为 1
→ T3↓ 时 ST0 在“错误模式语境”下被置 1
→ 下一拍阶段错乱
```

所以 **模式相关判断（含 SST0）统一以锁存后的 Q 为准**，和 `WRITE_REG` 等保持一致。

#### 4.3.4 SST0 与流程图上的 “SST0” 微操作

流程图里常在某拍列出控制信号：

```text
W1: SBUS, LPC, SST0, STOP, SHORT
```

这里的 `SST0` 就是“本拍要置位 ST0”的微操作。  
在 Verilog 里它不是 `output` 端口，而是内部 `wire`，只驱动 ST0 寄存器的置位条件——对数据通路不可见，对控制器状态机可见。

#### 4.3.5 一整拍里 Q / SST0 / ST0 的先后顺序

以“取指模式，装 PC 的启动拍”为例：

```text
假设：Q 已是 000，ST0=0，当前 W1

节拍内（组合逻辑）：
  INS_FETCH=1, ~ST0, W1
  → LPC=1, SBUS=1, SHORT=1, STOP=1
  → SST0 = 1          （因为 ST0==0 && Q==000 && W1）

T3 上升沿：
  PC 真正装入开关上的地址（LPC 生效）
  （ST0 此时仍是 0，本拍指令译码不会误触发）

T3 下降沿：
  看到 SST0=1 → ST0 <= 1
  同时可再次采样开关更新 Q

下一拍起：
  ST0=1，进入“正式执行/取指”阶段
```

**黄金顺序再记一次：**

```text
T3↑  先按“旧 ST0 / 旧 Q”完成数据写入
T3↓  再按 SST0 等条件更新 ST0 / Q，为下一拍服务
```

---

### 4.4 三者对照总表（建议背下来）

| | **Q** | **ST0** | **SST0** |
|--|-------|---------|----------|
| 是什么 | 3 bit **寄存器** | 1 bit **寄存器** | 1 bit **组合线** |
| 存什么 | 当前工作模式 | 前阶段(0)/后阶段(1) | 无存储，只是条件 |
| 谁驱动它 | T3↓ 采样 `SWC_SWB_SWA`；CLR 清 0 | T3↓：SST0 置 1；写寄存器 W2 可清 0；CLR 清 0 | `ST0==0` 且（模式×节拍） |
| 影响谁 | 全部模式译码、SST0、几乎所有分支 | 同一模式内的前后动作、指令是否允许译码 | 只影响 ST0 能否变 1 |
| 面板可见？ | 否（内部） | 否（内部，用其它灯推断） | 否 |
| 类比 | 电梯的“当前在哪一层模式” | “本层是进门大厅还是办公区” | “到点了，该从大厅进办公区”的门铃 |

---

### 4.5 为什么 Q/ST0 用下降沿，数据写入用上升沿？

```text
T3:  ___|‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾|___
         ↑ 上升沿              ↓ 下降沿
         数据通路写入           控制器换状态（Q / ST0）
         （用旧 Q/ST0）         （为下一拍准备）
```

这是故意错开的：

1. **上升沿**：按“本拍该做的事”（由旧 Q、旧 ST0 译出）把数据锁进 R0/PC/AR/IR……  
2. **下降沿**：才更新 Q/ST0，避免“写数据的半途状态机就变了”。

详见 `tolearn.md`。

---

### 4.6 小白常见误区

1. **“拨了开关就等于进了新模式”**  
   错。要等一次 QD 的 T3 下降沿锁进 **Q** 后，下一拍才算新模式。

2. **“ST0=1 就表示在执行指令”**  
   不准确。写/读存储器时 ST0=1 只表示已装好 AR。执行指令还要 `INS_FETCH && ST0`。

3. **“SST0 是输出到 CPU 的控制信号”**  
   在本实现里它是**内部置位条件**，不驱动数据通路；流程图写 SST0 是为了标明“本拍要置 ST0”。

4. **“CLR 只清数据寄存器”**  
   CLR 同时把 **Q、ST0、PCINC、LIR** 等控制器状态清干净。换模式/重跑前按 CLR 很重要。

5. **“读寄存器也要等 ST0”**  
   本设计读寄存器主要靠 W1/W2 切换显示组，SST0 条件里没有读寄存器分支。

## 5. 第二层：指令系统与执行流程

### 5.1 指令长什么样？

8 位指令，高 4 位是操作码：

```text
IR7 IR6 IR5 IR4 | IR3 IR2 | IR1 IR0
──── 操作码 ────   ─ Rd ─   ─ Rs ─
```

| 操作码 | 助记符 | 功能 | 拍数 |
|--------|--------|------|------|
| 0000 | NOP | 空操作 | 短（W1） |
| 0001 | ADD Rd,Rs | Rd ← Rd+Rs | 短 |
| 0010 | SUB Rd,Rs | Rd ← Rd−Rs | 短 |
| 0011 | AND Rd,Rs | Rd ← Rd∧Rs | 短 |
| 0100 | INC Rd | Rd ← Rd+1 | 短 |
| 0101 | LD Rd,[Rs] | Rd ← 内存[Rs] | **长（W1+W2）** |
| 0110 | ST Rs,[Rd] | 内存[Rd] ← Rs | **长** |
| 0111 | JC offset | C=1 则相对跳转 | 条件决定 |
| 1000 | JZ offset | Z=1 则相对跳转 | 条件决定 |
| 1001 | JMP [Rd] | PC ← Rd | **长** |
| 1011 | DEC Rd | Rd ← Rd−1 | 短（扩展） |
| 1100 | MOV Rd,Rs | Rd ← Rs | 短（扩展） |
| 1101 | CMP Rd,Rs | 比较，只改标志 | 短（扩展） |
| 1110 | STP | 停机 | 短 |
| 1111 | NOT Rd | Rd ← ¬Rd | 短（扩展） |

### 5.2 代码里如何“认出”指令？

```verilog
wire ADD = (IR7_IR4 == 4'b0001) & INS_FETCH & ST0;
wire LD  = (IR7_IR4 == 4'b0101) & INS_FETCH & ST0;
// ...
```

三个条件同时成立才算“正在执行 ADD”：

1. 操作码匹配  
2. 当前是取指模式（`INS_FETCH`）  
3. 已经过了启动拍（`ST0=1`）

这样手动模式（读写寄存器/存储器）下，即使 IR 里碰巧有值，也不会误触发指令分支。

### 5.3 取指模式的两阶段（跑程序前必须做的事）

**ST0=0，W1（启动拍）**

```text
SBUS + LPC + SST0 + STOP + SHORT
```

含义：把面板开关上的程序起始地址装进 PC，置 ST0=1，**本拍不取指**。

**ST0=1 之后**

按 `IR7~IR4` 译码执行。  
复位后 IR=00H（NOP），所以第一条“被执行”的总是 NOP——它会发出 `LIR+PCINC`，从而真正取到程序的第一条指令。这是 TEC-8 的约定，不是 bug。

### 5.4 短指令 vs 长指令

**短指令**（一拍搞定，W1 同时：运算 + 取下一条）：

```text
ADD 的 W1:
  S=1001, CIN, ABUS, DRW, LDZ, LDC   ← 做加法并写回
  LIR, PCINC, SHORT                  ← 同时取下一条
```

**长指令**（两拍，取指放在 W2）：

```text
LD:
  W1: M, S=1010, ABUS, LAR           ← 把地址送进 AR
  W2: MBUS, DRW, LIR, PCINC          ← 读内存写寄存器 + 取指

ST:
  W1: M, S=1111, ABUS, LAR           ← 地址 → AR
  W2: M, S=1010, ABUS, MEMW, LIR, PCINC  ← 数据写入内存 + 取指
```

**条件跳转**：

```text
JC / JZ:
  条件不成立 → 等同短指令（W1 直接取下一条）
  条件成立   → W1: PCADD；W2: LIR+PCINC
```

代码用 `SHORT_INSTR` 汇总所有“只需 W1”的情况：

```verilog
wire SHORT_INSTR = NOP | ADD | SUB | AND_ | INC | MOV | CMP | NOT_ | DEC
                 | (JC & ~C) | (JZ & ~Z);
assign SHORT = ... | (SHORT_INSTR & W1);
```

## 6. 第三层：控制信号是怎么“算”出来的

核心思想：**每个控制信号 = 所有需要它的“模式×指令×节拍”条件的或（OR）**。

### 6.1 例子：DRW（写寄存器使能）

```verilog
assign DRW = (WRITE_REG & (W1|W2))          // 手动写寄存器两拍都写
           | ((ADD|SUB|AND_|INC|MOV|NOT_|DEC) & W1)  // 运算类短指令
           | (LD & W2);                     // LD 在第二拍写回
```

读法：  
“只要处于这些情况之一，就允许写通用寄存器。”

### 6.2 例子：总线三选一

```verilog
assign SBUS = (WRITE_REG & (W1|W2))
            | (READ_MEM & W1 & ~ST0)
            | (WRITE_MEM & W1)
            | (INS_FETCH & ~ST0 & W1);   // 启动时装 PC

assign MBUS = (READ_MEM & W1 & ST0) | (LD & W2);

assign ABUS = ((ADD|SUB|AND_|INC|LD|ST|JMP|MOV|CMP|NOT_|DEC) & W1)
            | (ST & W2);
```

同一时刻应只有一个来源有效，否则总线冲突。

### 6.3 例子：ALU 功能 S/M/CIN

对照 `74ls181.md` 的正逻辑表：

| 指令 | M | S | CIN | 实现的运算 |
|------|---|---|-----|------------|
| ADD | 0 | 1001 | 1 | A+B |
| SUB | 0 | 0110 | 0 | A−B |
| AND | 1 | 1011 | — | A∧B |
| INC | 0 | 0000 | 0* | 见源码与流程图 |
| MOV | 1 | 1010 | — | F=A（直通） |
| NOT | 1 | 0000 | — | F=¬A |
| DEC | 0 | 1111 | 1 | A−1 |
| CMP | 0 | 0110 | 0 | 同 SUB，但**无 DRW** |

```verilog
assign M   = ((AND_|LD|ST|JMP|MOV|NOT_) & W1) | (ST & W2);
assign CIN = (ADD | DEC) & W1;
assign S[3] = ((ADD|AND_|LD|JMP|MOV|DEC) & W1) | (ST & (W1|W2));
// S[2]、S[1]、S[0] 同理……
```

CMP 故意不加 `DRW`，所以只更新标志、不破坏 Rd——这是和 SUB 的唯一关键差别。

### 6.4 标志位

```verilog
assign LDZ = (ADD|SUB|AND_|INC|MOV|CMP|NOT_|DEC) & W1;  // 结果是否为 0
assign LDC = (ADD|SUB|INC|CMP|DEC) & W1;                 // 是否产生进位
```

逻辑运算一般不更新 C；MOV 更新 Z 便于后续判断。

### 6.5 手动模式信号（以写寄存器为例）

四个 QD 依次写 R0→R1→R2→R3，靠 `ST0 + W1/W2` 拼出不同的 `SEL`：

| 次序 | ST0 | 节拍 | SEL | 目标 |
|------|-----|------|-----|------|
| 1 | 0 | W1 | 0011 | R0 |
| 2 | 0 | W2 | 0100 | R1（结束时 ST0→1） |
| 3 | 1 | W1 | 1001 | R2 |
| 4 | 1 | W2 | 1110 | R3（结束时 ST0→0） |

对应：

```verilog
assign SEL[3] = (WRITE_REG & (W1|W2) & ST0) | (READ_REG & W2);
assign SEL[2] = (WRITE_REG & W2);
assign SEL[1] = (WRITE_REG & ((W1 & ~ST0) | (W2 & ST0))) | (READ_REG & W2);
assign SEL[0] = (WRITE_REG & W1) | (READ_REG & (W1|W2));
```

## 7. 关键设计：PCINC / LIR 时序化（解决吞指令）

### 7.1 什么是“吞指令”？

流水线把“取指”叠进执行拍，需要**同时**拉高：

- `LIR`：把当前 PC 指向的指令装进 IR  
- `PCINC`：PC ← PC+1  

正常顺序必须是 **先锁存当前指令，再 PC+1**。  
若因布线延时变成 **PC 先 +1，再锁存**，IR 拿到的就是**下下条**——中间那条被“吞掉”。

### 7.2 组合逻辑为什么容易反序？

早期写法用纯组合 `assign`/`always @(*)` 同时产生两路电平。  
两路门延时/布线不同 → 到达 PC 与 IR 的时刻有先后 → 在 T3 采样窗附近可能反序。  
顺序式控制器取指独占一拍、路径简单，问题被裕量盖住；流水线合并后问题暴露。

### 7.3 本代码的修法

把 `PCINC`、`LIR` 改成 **同一表达式、同一时钟沿锁存的寄存器**：

```verilog
wire FETCH;
assign FETCH = (INS_FETCH && ST0) && (
    (W1 && ( /* 所有短指令操作码，含 JC/JZ 不跳 */ )) ||
    (W2 && ( /* LD/ST、JC/JZ 跳转成立、JMP */ ))
);

always @(posedge T3 or negedge CLR) begin
    if (!CLR) begin
        PCINC <= 1'b0;
        LIR   <= 1'b0;
    end else begin
        PCINC <= FETCH;
        LIR   <= FETCH;   // 与 PCINC 完全相同
    end
end
```

效果：

1. 两路从**同一上升沿**一起跳变，严格同步  
2. 整个 T3 高电平期间两者稳定同为高  
3. 在响应沿（通常为 T3 下降沿）同沿采样 → 天然“先取当前、再 +1”

### 7.4 何时取指（速查）

| 情况 | 取指节拍 |
|------|----------|
| 启动拍 `INS_FETCH && !ST0 && W1` | **不取指**（只装 PC） |
| 短指令、JC/JZ 不跳 | **W1** |
| LD / ST / JMP / JC·JZ 跳转成立 | **W2** |
| STP、手动模式 | **不取指** |

更细的时序图见 `流水线吞指令问题记录.md`。

## 8. 一条指令走完全程（ADD 例子）

假设已进入取指模式且 `ST0=1`，当前 IR 为 `ADD R0, R1`（操作码 0001）。

**W1 期间组合逻辑拉高：**

```text
S = 1001, CIN=1, M=0     → ALU 做 R0+R1
ABUS                      → 结果上总线
DRW                       → 允许写回 R0
LDZ, LDC                  → 准备更新标志
SHORT                     → 只要这一拍
FETCH=1 → T3 上升沿后 PCINC=LIR=1
```

**时间轴：**

```text
W1: T1/T2  ── ALU 算完，结果稳定在总线上
    T3↑   ── R0 写入结果；Z/C 更新；PCINC/LIR 寄存器变为 1
    T3↓   ── IR 锁存下一条指令；PC+1；控制器状态可切换
```

于是：**执行本条 + 取下一条** 在同一机器周期完成——这就是流水线的含义。

## 9. 数据通路在控制器眼中的样子

控制器不实现 ALU/寄存器本身，只“发号施令”。一条运算指令的数据流：

```text
        Rs ──┐
             ├─► ALU(S,M,CIN) ─ABUS─► 总线 ─DRW─► Rd
        Rd ──┘                      │
                                    ├─LDZ─► Z
                                    └─LDC─► C
```

访存指令（LD）：

```text
W1: Rs ─ALU直通─ABUS─► LAR ──► AR
W2: 存储器[AR] ─MBUS─► 总线 ─DRW─► Rd
```

启动程序：

```text
开关 SD ─SBUS─► 总线 ─LPC─► PC
之后 ST0=1，靠 NOP/各指令末尾的 LIR 不断取指执行
```

## 11. 设计原则小结（记住这 7 条就够）

1. **控制器 = 状态 + 译码**：Q/ST0 管状态，assign 管译码。  
2. **组合出控制电平，时序沿才真正改数据**。  
3. **T3 上升沿写数据通路，下降沿换控制器状态** —— 故意错开。  
4. **模式必须先锁存再使用** —— 不要信任正在抖动的开关。  
5. **流水线 = 执行与取指重叠**；短指令 W1 取指，长指令 W2 取指。  
6. **PCINC 与 LIR 必须同步** —— 用同一 FETCH、同一时钟沿寄存器化。  
7. **每个 assign 都是流程图条件的布尔或** —— 改指令流程时，同步改对应信号的条件项。

## 12. 术语速查表

| 术语 | 一句话 |
|------|--------|
| 硬布线控制器 | 用组合逻辑直接产生微操作，不用微程序 |
| 流水线指令周期 | 取指叠在执行拍，缩短平均周期 |
| 微操作 | 一个控制信号对应的原子动作（如 DRW、LPC） |
| 机器周期 / 节拍 | W1、W2；内部再分 T1~T3 |
| QD | 面板单步按钮，一次推进一个 W |

| Q | 锁存后的工作模式 |
| IR | 指令寄存器，现指令操作码来自 IR7~IR4 |

| 吞指令 | LIR/PCINC 反序导致取到下下条 |
| SHORT | 本周期在 W1 结束，不进 W2 |
| STOP | 停机或单步等待下一次 QD |
