## 二、需对照 TEC-8 ALU 功能表确认的问题

### 问题 #1 / #5：NOT 指令 `S` 编码缺失（已修复）

**位置**：`pipelined_hardwired_controller.v`，指令 `4'b1101`（NOT）的 W1 分支。

**当前代码**（已优化为短周期，但 `S` 仍未设置）：

```verilog
4'b1101: begin // NOT：执行按位取反并写回寄存器。
    if (W1) begin // 短周期：取反结果经 ABUS 写回寄存器，同时取下一条指令。
        SHORT = 1'b1;
        PCINC = 1'b1;
        LIR = 1'b1;
        DRW = 1'b1;
        ABUS = 1'b1;
        LDZ = 1'b1;
        M = 1'b1;
        // 注意：未对 S 赋值，S 保持默认 4'b0000
    end
end
```

**问题描述**：

NOT 是逻辑运算（`M = 1`），按位取反需要 ALU 提供专门的"取反"功能码。
当前代码只设了 `M = 1`，没有设置 `S`。

> **更新注记（2026-07-08）**：主控制组合逻辑块开头的默认清零赋值（`S = 4'b0000` 等
> 20 行）已被删除（经实测确认不必要）。因此 NOT 分支未赋值的 `S` 不再有稳定默认值
> `4'b0000`，而是会综合为锁存器、保持上一拍指令设置的 `S`。即 NOT 的实际行为现在
> 取决于上一条指令的 `S` 编码，原结论“等价于 `MOV A→A`（基于 `S=0000`）”的前提已
> 不再恒成立。若实测 NOT 行为正常，需核对其前置指令的 `S` 值后再下结论。

**已查证依据**：

- 标准 74LS81 正逻辑（active-high）功能表中，`M=1, S=0000` 对应 **`F = A`**
  （传送），**不是取反**；取反 `F = /A` 出现在 active-low 列。
- TEC-8 实验指导书摘选表（多份实验报告一致）中，`S=0000` 在逻辑模式下同样
  标注为 `F = A`。
- 代码中 INC 用 `S=0000` 走算术 `A+1`、AND 用 `S=1011` 走逻辑与，均与该表
  吻合，进一步佐证该表可信。

**结论**：`S = 0000` 在 TEC-8 中是**传送 A**，不是取反。因此 NOT 修复前等价于
`MOV A→A`，只更新了 Z 标志，**完全没有取反功能**。**确认为 Bug。**

**修复依据**：TEC-8 使用 74181 的 active-high（正逻辑）功能表。在该表的逻辑模式
（M=H）列中，`S=0100` 对应 `F = /A`（按位取反）。该结论已与代码中已验证的多个
逻辑指令交叉自洽：

| 指令 | S（M=1） | active-high 表功能 | 代码期望 | 自洽 |
| --- | --- | --- | --- | --- |
| MOV/OUT/LD/ST | 1010 | F = B | 传 B | ✓ |
| AND | 1011 | F = A·B | 与 | ✓ |
| JMP | 1111 | F = A | 传 A | ✓ |

此外，注意 active-low 列中 `S=0000` 写作 `/A`，TEC-8 摘选表未列入取反项是因为
只摘录了部分常用功能；而 active-high 列的取反功能为 `S=0100`。

**已应用的修复**（`pipelined_hardwired_controller.v`，NOT 分支）：

```verilog
M = 1'b1;
S = 4'b0100; // active-high 74181 逻辑模式: F = /A
```

> **待实测确认**：修改后请在 TEC-8 上用 `0FH/00H/FFH` 三组初值实测 NOT，确认
> R1 写回值分别为 `F0H/FFH/00H`，Z 标志在结果为 00H 时为 1。

### 问题 #6：DEC 指令 ALU 编码 `S / CIN`（基本无问题，建议二次确认）

**位置**：`pipelined_hardwired_controller.v`，指令 `4'b1111`（DEC）的 W1 分支。

**当前代码**（已优化为短周期，ALU 编码未变）：

```verilog
4'b1111: begin // DEC：执行自减并写回寄存器，同时更新 Z/C。
    if (W1) begin // 短周期：自减结果写回寄存器并更新标志位，同时取下一条指令。
        SHORT = 1'b1;
        PCINC = 1'b1;
        LIR = 1'b1;
        DRW = 1'b1;
        CIN = 1'b1;
        ABUS = 1'b1;
        LDZ = 1'b1;
        LDC = 1'b1;
        S = 4'b1111;
    end
end
```

**问题描述**：

DEC（自减，即 `A - 1`）的常用实现思路有两种：

1. **减法思路**：`A - 1 = A + (-1 的补码) = A + 1111 + 0`，使用减法功能码
   （参考 SUB：`S = 0110`），`CIN = 0`。
2. **加法思路**：`A - 1 = A + 1111 + 1 = A + (1111 + 1) 溢出回 0000`，使用
   加法功能码（参考 ADD：`S = 1001`），`CIN = 1`。

当前代码使用 `S = 1111` + `CIN = 1`。

**已查证依据**：

- TEC-8 摘选表中 `S=1111, M=0, CIN=1`（即 CN=1，无进位）对应 **`F = A - 1`**，
  与 DEC 语义吻合。
- 代码中 ADD 用 `CIN=1`、INC 用默认 `CIN=0`，反推 TEC-8 的 `CIN` 等价于 74181
  的 `/Cn`（低有效），与摘选表"CN=1 无进位"的标注一致。

**结论**：`S=1111, CIN=1` 在 TEC-8 中确实实现 `A - 1`，**基本确认不是 Bug**。
`LDC` 的借位语义也与 SUB 一致，合理。

**建议**：仍建议查完整表二次确认 `S=1111` 的算术运算定义。
