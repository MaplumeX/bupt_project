## 一、已修复 Bug 归档

### Bug #2：取指阶段 ST0=0 的 W2 时序隐患（已修复）

**位置**：`pipelined_hardwired_controller.v`，`INS_FETCH` 分支 `ST0=0` 段。

**历史问题描述**：

原实现在 `ST0=0` 段同时安排了 W1（装 PC）和 W2（取第一条指令 `LIR+PCINC`），
但 `SST0` 在取指模式用 W1 触发，`ST0` 在 T3 下降沿更新。若 T3 下降沿位于
W1→W2 切换处，则 W2 期间 `ST0` 已变为 1，主控制逻辑进入 `else`（执行分支）
按 `IR7_IR4` 译码，而非执行流程图要求的 `LIR + PCINC`。原本只靠复位后
IR=00H=NOP 的“巧合性正确”掩盖了该问题。

**修复方案**：改用文档「4.5 任意修改 PC 指针」的单拍短周期写法。

- 给 `ST0=0` 的 W1 拉 `SHORT`，把本机器周期压成单拍，不再有 W2；
- `ST0=0` 段只做 `SBUS + LPC + SST0 + STOP + SHORT`，不在本周期内取指；
- 第一条指令的取指交给下一周期（`ST0=1`）的自然 NOP 流程：复位后 IR=00H=NOP，
  NOP 分支会输出 `LIR+PCINC`，完成初始进入程序。

修复后代码：

```verilog
INS_FETCH: begin
    if (!ST0) begin
        if (W1) begin
            SBUS  = 1'b1;
            LPC   = 1'b1;
            STOP  = 1'b1;
            SHORT = 1'b1;
        end
    end
    else begin
        case (IR7_IR4)
            4'b0000: begin // NOP：复位后第一条指令必为 00H=NOP，由此完成首次取指
                if (W1) begin
                    SHORT = 1'b1;
                    PCINC = 1'b1;
                    LIR   = 1'b1;
                end
            end
            ...
```

`SST0` 仍保持 W1 触发，但因 `SHORT` 压成了单拍，不再存在"W2 时 ST0 已翻"的
冲突。

## 二、需对照 TEC-8 ALU 功能表确认的问题

### 问题 #1 / #5：NOT 指令 `S` 编码缺失（确认为 Bug，暂不修复）

**位置**：`pipelined_hardwired_controller.v`，指令 `4'b1101`（NOT）的 W1 分支。

**当前代码**（已优化为短周期，但 `S` 仍未设置）：

```verilog
4'b1101: begin // NOT：执行按位取反并写回寄存器。
    if (W1) begin // 短周期：取反结果经 ABUS 写回寄存器，同时取下一条指令。
        SHORT = 1'b1;
        PCINC = 1'b1;
        LIR = 1'b1;
        DRW = 1'b1;
        ABUS = 1'b1;
        LDZ = 1'b1;
        M = 1'b1;
        // 注意：未对 S 赋值，S 保持默认 4'b0000
    end
end
```

**问题描述**：

NOT 是逻辑运算（`M = 1`），按位取反需要 ALU 提供专门的"取反"功能码。
当前代码只设了 `M = 1`，没有设置 `S`，于是 `S` 保持组合逻辑块开头的默认值
`4'b0000`。

**已查证依据**：

- 标准 74LS81 正逻辑（active-high）功能表中，`M=1, S=0000` 对应 **`F = A`**
  （传送），**不是取反**；取反 `F = /A` 出现在 active-low 列。
- TEC-8 实验指导书摘选表（多份实验报告一致）中，`S=0000` 在逻辑模式下同样
  标注为 `F = A`。
- 代码中 INC 用 `S=0000` 走算术 `A+1`、AND 用 `S=1011` 走逻辑与，均与该表
  吻合，进一步佐证该表可信。

**结论**：`S = 0000` 在 TEC-8 中是**传送 A**，不是取反。因此 NOT 当前等价于
`MOV A→A`，只更新了 Z 标志，**完全没有取反功能**。**确认为 Bug。**

**为何暂不修复**：TEC-8 摘选表未列出取反对应的 `S` 编码，需查 TEC-8 **完整**
ALU 功能表确认"按位取反"对应的 `S` 值后再补入。不能直接套用标准 74181 表的
`S=0000 → /A`（那是 active-low 列，TEC-8 用正逻辑）。

**修复示例**（待确认正确编码后填入）：

```verilog
M = 1'b1;
S = 4'b????;   // 以 TEC-8 完整 ALU 功能表给出的取反编码为准
```

### 问题 #6：DEC 指令 ALU 编码 `S / CIN`（基本无问题，建议二次确认）

**位置**：`pipelined_hardwired_controller.v`，指令 `4'b1111`（DEC）的 W1 分支。

**当前代码**（已优化为短周期，ALU 编码未变）：

```verilog
4'b1111: begin // DEC：执行自减并写回寄存器，同时更新 Z/C。
    if (W1) begin // 短周期：自减结果写回寄存器并更新标志位，同时取下一条指令。
        SHORT = 1'b1;
        PCINC = 1'b1;
        LIR = 1'b1;
        DRW = 1'b1;
        CIN = 1'b1;
        ABUS = 1'b1;
        LDZ = 1'b1;
        LDC = 1'b1;
        S = 4'b1111;
    end
end
```

**问题描述**：

DEC（自减，即 `A - 1`）的常用实现思路有两种：

1. **减法思路**：`A - 1 = A + (-1 的补码) = A + 1111 + 0`，使用减法功能码
   （参考 SUB：`S = 0110`），`CIN = 0`。
2. **加法思路**：`A - 1 = A + 1111 + 1 = A + (1111 + 1) 溢出回 0000`，使用
   加法功能码（参考 ADD：`S = 1001`），`CIN = 1`。

当前代码使用 `S = 1111` + `CIN = 1`。

**已查证依据**：

- TEC-8 摘选表中 `S=1111, M=0, CIN=1`（即 CN=1，无进位）对应 **`F = A - 1`**，
  与 DEC 语义吻合。
- 代码中 ADD 用 `CIN=1`、INC 用默认 `CIN=0`，反推 TEC-8 的 `CIN` 等价于 74181
  的 `/Cn`（低有效），与摘选表"CN=1 无进位"的标注一致。

**结论**：`S=1111, CIN=1` 在 TEC-8 中确实实现 `A - 1`，**基本确认不是 Bug**。
`LDC` 的借位语义也与 SUB 一致，合理。

**建议**：仍建议查完整表二次确认 `S=1111` 的算术运算定义。
