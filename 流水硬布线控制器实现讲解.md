# TEC-8 流水硬布线控制器实现讲解

---

## 1. 先用一句话理解这个项目

这个项目实现的是 TEC-8 实验平台上的一个**硬布线控制器**。

它的作用可以理解成：

> 根据当前模式、当前节拍、当前指令和标志位，自动点亮一组控制信号，让 TEC-8 的寄存器、ALU、存储器、PC、IR 等部件按正确顺序工作。

换句话说，它不是负责“算加法”或“存数据”的部件，而是负责告诉各个硬件部件：

- 现在谁把数据送到总线？
- 现在数据写入哪个寄存器？
- PC 要不要加 1？
- IR 要不要装入下一条指令？
- 存储器要不要写？
- ALU 要执行加法、减法、与运算，还是传送？
- 条件跳转时到底跳不跳？

可以把控制器想象成一名“交通指挥员”。TEC-8 的数据通路是道路和车辆，控制器负责在每个时刻给出红绿灯。

---

## 2. TEC-8 和控制器分别是什么

### 2.1 TEC-8 实验平台大致有什么

TEC-8 是一个教学用的简单计算机实验平台。它通常包含：

| 部件 | 可以粗略理解为 | 作用 |
| --- | --- | --- |
| 数据开关 SD7~SD0 | 手动输入 | 给总线输入地址、数据或程序起始地址 |
| 通用寄存器 R0~R3 | 小型存储单元 | 暂存运算数据 |
| ALU | 计算器 | 做加法、减法、与、传送等操作 |
| 存储器 | 内存 | 存放程序和数据 |
| PC | 程序计数器 | 指向下一条要取的指令地址 |
| IR | 指令寄存器 | 保存当前正在执行的指令 |
| AR | 地址寄存器 | 保存当前访问存储器的地址 |
| C / Z 标志 | 状态灯 | 记录进位、结果是否为零 |
| 控制信号灯 | 控制线 | 控制上述部件是否工作 |

### 2.2 控制器做什么

控制器输入一些“当前情况”：

```text
模式开关 SWC/SWB/SWA
节拍 W1/W2
时钟相关信号 T3
复位 CLR
当前指令高 4 位 IR7~IR4
标志位 C/Z
```

然后输出一堆“控制命令”：

```text
SBUS、MBUS、ABUS、DRW、MEMW、LIR、PCINC、LAR、LPC、STOP、SHORT、S、M、SEL ...
```

这些输出信号会直接接到 TEC-8 的数据通路上，决定每一拍硬件怎么动。

## 3. 这个项目实现了什么功能

项目中的核心文件是：

```text
pipelined_hardwired_controller.v
```

它实现了一个 Verilog 模块：

```verilog
module pipelined_hardwired_controller (...);
```

这个模块主要实现了三类功能。

---

### 3.1 功能一：支持 TEC-8 面板的 5 种工作模式

TEC-8 通过三个模式开关 `SWC_SWB_SWA` 选择当前工作模式。

| SWC SWB SWA | 代码中对应信号 | 功能 |
| --- | --- | --- |
| `100` | `WRITE_REG` | 手动写寄存器 |
| `011` | `READ_REG` | 手动读寄存器 |
| `010` | `READ_MEM` | 手动读存储器 |
| `001` | `WRITE_MEM` | 手动写存储器 |
| `000` | `INS_FETCH` | 取指 / 执行程序 |

也就是说，这个控制器不仅能运行程序，还能完成实验前常见的手动操作：

- 把数据写进寄存器；
- 从寄存器读出数据；
- 把程序或数据写进内存；
- 从内存读出数据；
- 从指定地址开始执行程序。

---

### 3.2 功能二：支持指令取指和执行

进入 `000` 取指/执行模式后，控制器会根据 `IR7_IR4` 译码。

`IR7_IR4` 是当前指令的高 4 位，也就是操作码。代码中覆盖了 16 种 4 位操作码：

| IR7~IR4 | 指令 | 作用 |
| --- | --- | --- |
| `0000` | NOP | 空操作，只取下一条指令 |
| `0001` | ADD | 加法，写回寄存器 |
| `0010` | SUB | 减法，写回寄存器 |
| `0011` | AND | 按位与，写回寄存器 |
| `0100` | INC | 自增 |
| `0101` | LD | 从存储器读数到寄存器 |
| `0110` | ST | 把寄存器数据写入存储器 |
| `0111` | JC | C=1 时条件跳转 |
| `1000` | JZ | Z=1 时条件跳转 |
| `1001` | JMP | 无条件跳转 |
| `1010` | OUT | 将寄存器内容输出到总线 |
| `1011` | MOV | 寄存器间传送 |
| `1100` | CMP | 比较，只更新标志位 |
| `1101` | NOT | 取反，当前代码存在待确认问题 |
| `1110` | STP | 停机 |
| `1111` | DEC | 自减 |

其中基础实验文档里主要列出 ADD、SUB、AND、INC、LD、ST、JC、JZ、JMP、STOP 等指令；代码实际还扩展实现了 OUT、MOV、CMP、NOT、DEC 等操作码。

---

### 3.3 功能三：把设计映射到 EPM7128 CPLD 引脚

项目使用的器件是：

```text
Altera MAX7000S / EPM7128SLC84-15
```

引脚约束在 `pipelined_hardwired_controller.qsf` 和 `作为硬连线控制器时的EPM7128引脚规定.md` 中。

例如：

| 信号 | 引脚 |
| --- | --- |
| `CLR` | PIN_1 |
| `T3` | PIN_83 |
| `W1` | PIN_12 |
| `W2` | PIN_15 |
| `ABUS` | PIN_40 |
| `SBUS` | PIN_41 |
| `MBUS` | PIN_44 |
| `MEMW` | PIN_27 |
| `STOP` | PIN_28 |
| `LIR` | PIN_29 |
| `PCINC` | PIN_21 |
| `SELCTL` | PIN_52 |

所以，这份 Verilog 不是单纯“仿真玩具”，而是可以综合后下载到 TEC-8 的 CPLD 上，直接驱动实验板。

## 4. 为什么叫“硬布线控制器”

控制器常见有两种实现方式：

| 类型 | 思路 | 类比 |
| --- | --- | --- |
| 微程序控制器 | 用一张“微指令表”一步步查表输出控制信号 | 像照着菜谱做菜 |
| 硬布线控制器 | 用逻辑电路直接根据输入组合出控制信号 | 像固定线路的开关电路 |

本项目是**硬布线控制器**，因为代码里没有微指令 ROM，而是直接写逻辑判断：

```verilog
如果当前是 ADD 指令，并且处于 W1 节拍：
    打开 ABUS
    打开 DRW
    打开 LDZ
    打开 LDC
    设置 ALU 功能 S
    同时 PCINC、LIR 取下一条指令
```

也就是说，每条指令在每个节拍要输出哪些控制信号，都是由 Verilog 中的 `case` 和 `if` 直接决定的。

### 为什么这样实现

因为 TEC-8 实验中的指令数量少、节拍简单，用硬布线方式有几个好处：

1. **直观**：每条指令对应哪些控制信号，在代码里直接能看到。
2. **速度快**：不需要查微程序 ROM，输入变化后组合逻辑直接产生输出。
3. **适合 CPLD**：EPM7128 这类器件资源有限，硬布线逻辑更贴近实验要求。
4. **便于和流程图对应**：实验指导书中的流程图本来就是“某模式 + 某节拍 → 某些控制信号”。

---

## 5. 为什么叫“流水”

这里的“流水”不要理解成现代 CPU 那种复杂的五级流水线。

这个项目里的“流水”更接近：

> 在执行当前指令的同时，顺便取下一条指令。

你会在很多短指令中看到类似输出：

```text
执行本指令需要的信号
+
PCINC
LIR
SHORT
```

其中：

| 信号 | 含义 |
| --- | --- |
| `PCINC` | PC 加 1，指向下一条指令 |
| `LIR` | 把下一条指令装入 IR |
| `SHORT` | 告诉节拍发生器：这条指令是短周期，一拍完成 |

例如 ADD 指令在 W1 里同时做两件事：

1. ALU 执行加法，结果写回寄存器；
2. PC 加 1，并把下一条指令装入 IR。

这样一来，当前指令执行完，下一条指令已经准备好了，程序就能连续向前跑。

### 为什么这样实现

因为 TEC-8 的指令很简单，很多指令一拍就能完成。把“执行”和“取下一条指令”放在同一个节拍里，可以减少等待时间，让控制流程更紧凑。

不过要注意：这不是完整 CPU 流水线，没有复杂的冒险检测、暂停转发等机制。

## 6. 代码整体结构

`pipelined_hardwired_controller.v` 可以分成 4 个部分理解。

---

### 6.1 第一部分：输入输出端口

模块输入包括：

```verilog
input wire [2:0] SWC_SWB_SWA
input wire [3:0] IR7_IR4
input wire CLR
input wire T3
input wire W1, W2
input wire C, Z
```

模块输出包括：

```verilog
SELCTL, ABUS, SBUS, MBUS, M, CIN, DRW, LDZ, LDC,
MEMW, ARINC, PCINC, PCADD, LPC, LAR, LIR, STOP, SHORT,
S, SEL
```

输入代表“当前情况”，输出代表“控制命令”。

---

### 6.2 第二部分：锁存工作模式 Q

代码中有一个内部寄存器：

```verilog
reg [2:0] Q;
```

它用来保存当前工作模式。

模式不是直接用 `SWC_SWB_SWA`，而是在 `T3` 下降沿锁存进 `Q`：

```verilog
always @(negedge T3 or negedge CLR)
```

### 为什么要锁存模式

因为面板开关是人手拨动的，可能出现抖动或中间状态。

如果控制器直接使用开关输入，那么你拨动开关的一瞬间，控制信号可能乱跳。

所以代码选择：

> 只在固定时刻 `T3` 下降沿采样一次模式开关，然后整个周期都使用锁存后的 `Q`。

这样更稳定，也更符合 TEC-8 的时序。

---

### 6.3 第三部分：ST0 阶段标志

代码中还有一个内部寄存器：

```verilog
reg ST0;
```

它是一个“阶段标志”。

可以粗略理解为：

| ST0 | 含义 |
| --- | --- |
| `0` | 当前处于第一阶段，例如装地址、装 PC、写前两个寄存器 |
| `1` | 当前处于第二阶段，例如连续读写内存、正式执行指令、写后两个寄存器 |

不同模式下，`ST0` 的意义略有不同。

例如读存储器模式：

| ST0 | 操作 |
| --- | --- |
| `0` | 把数据开关上的地址装入 AR |
| `1` | 从当前 AR 指向的内存读数据，并让 AR 加 1 |

写存储器模式：

| ST0 | 操作 |
| --- | --- |
| `0` | 把数据开关上的地址装入 AR |
| `1` | 把数据开关上的数据写入内存，并让 AR 加 1 |

取指模式：

| ST0 | 操作 |
| --- | --- |
| `0` | 初始化 PC，准备开始执行程序 |
| `1` | 正式按照 IR 中的指令执行 |

### 为什么需要 ST0

因为同一个模式下，经常不是一步就结束。

例如写存储器不能只做一件事，它至少要分成：

1. 先告诉机器“我要写哪个地址”；
2. 再告诉机器“我要往这个地址写什么数据”。

如果没有 `ST0`，控制器就无法记住自己现在到底在第几步。

---

### 6.4 第四部分：主控制组合逻辑

主控制逻辑在：

```verilog
always @(*) begin
    ...
end
```

这个部分是整个控制器的核心。

它的写法是：

1. 先把所有输出信号默认清零；
2. 根据当前模式选择一个分支；
3. 在分支里根据 `W1/W2`、`ST0`、`IR7_IR4`、`C/Z` 拉高需要的控制信号。

代码结构大致是：

```text
先清零所有输出

如果是写寄存器模式：
    根据 W1/W2 和 ST0 输出 SBUS、DRW、SEL...

如果是读寄存器模式：
    输出 SELCTL、SEL、STOP...

如果是读存储器模式：
    ST0=0 装地址
    ST0=1 读数据并 AR+1

如果是写存储器模式：
    ST0=0 装地址
    ST0=1 写数据并 AR+1

如果是取指/执行模式：
    ST0=0：单拍短周期装入 PC 起始地址
    ST0=1：根据 IR7~IR4 执行具体指令
```

### 为什么先把所有输出清零

这是 Verilog 组合逻辑里很重要的写法。

如果不先给默认值，某些分支没有给某个输出赋值时，综合工具可能会推断出锁存器，导致行为不稳定。

因此代码先写：

```verilog
SELCTL = 1'b0;
ABUS = 1'b0;
SBUS = 1'b0;
...
```

然后只在需要的时候把某些信号置 1。

这样逻辑更安全、更清晰。

---

## 7. 五种模式是怎么实现的

---

### 7.1 写寄存器模式 `100`

目标：把数据开关 `SD7~SD0` 上的值写入 R0~R3。

代码输出：

```text
SBUS = 1      数据开关送总线
SELCTL = 1    使用手动寄存器选择
DRW = 1       写寄存器
STOP = 1      单步/暂停，方便人工操作
SEL = ...     选择具体寄存器
```

它通过 `W1/W2` 和 `ST0` 组合出不同的 `SEL`，依次选择不同寄存器。

大致流程：

| 阶段 | SEL | 作用 |
| --- | --- | --- |
| ST0=0, W1 | `0011` | 写 R0 |
| ST0=0, W2 | `0100` | 写 R1，然后 ST0 变 1 |
| ST0=1, W1 | `1001` | 写 R2 |
| ST0=1, W2 | `1110` | 写 R3，然后 ST0 回 0 |

### 为什么这样实现

TEC-8 只有一组数据开关，但有多个寄存器。控制器必须通过 `SEL` 选择“这次写哪个寄存器”。

用 `ST0 + W1/W2` 可以在连续按单步时依次写入 R0、R1、R2、R3。

---

### 7.2 读寄存器模式 `011`

目标：把寄存器内容读到总线上，方便从数据灯观察。

代码输出：

```text
SELCTL = 1
STOP = 1
SEL = 指定组合
```

注意它不会打开 `DRW`，因为读寄存器不能写坏原来的数据。

### 为什么这样实现

读寄存器只是观察数据，不应该改变机器状态。所以只选择寄存器输出，不写入任何寄存器或存储器。

---

### 7.3 读存储器模式 `010`

目标：先指定地址，再连续读内存。

流程：

| ST0 | 控制信号 | 含义 |
| --- | --- | --- |
| `0` | `SBUS + LAR` | 数据开关上的地址装入 AR |
| `1` | `MBUS + ARINC` | 存储器数据送总线，同时 AR 加 1 |

也就是说，第一次告诉机器“从哪里开始读”，后面每按一次就读下一个地址。

### 为什么这样实现

手动读内存时，通常不是只读一个字节，而是从某个起始地址开始连续查看多个单元。

所以设计成：

1. 第一拍装入起始地址；
2. 后续每拍读出当前地址内容，并自动 `AR+1`。

这样操作更方便。

---

### 7.4 写存储器模式 `001`

目标：先指定地址，再连续写内存。

流程：

| ST0 | 控制信号 | 含义 |
| --- | --- | --- |
| `0` | `SBUS + LAR` | 数据开关上的地址装入 AR |
| `1` | `SBUS + MEMW + ARINC` | 数据开关上的数据写入内存，同时 AR 加 1 |

### 为什么这样实现

写程序到 TEC-8 内存时，通常要连续写很多字节，例如：

```text
00H: 00
01H: 13
02H: 23
03H: E0
```

如果每写一个字节都重新输入地址会很麻烦。

所以控制器让 AR 自动加 1。你只需要改变数据开关，然后按单步，就能连续写入一段程序。

---

### 7.5 取指 / 执行模式 `000`

这是最核心的模式。

它分为两个阶段。

#### 阶段一：ST0 = 0，初始化 PC

| 节拍 | 控制信号 | 含义 |
| --- | --- | --- |
| W1 | `SBUS + LPC + STOP + SHORT` | 把数据开关上的程序起始地址装入 PC，单拍短周期 |

也就是说，执行程序前，你先用数据开关指定程序从哪里开始。

> 本拍不取指（无 `LIR`）。第一条指令的取指由下一周期（`ST0=1`）的自然 NOP
> 流程完成：复位后 IR=00H=NOP，NOP 分支输出 `LIR+PCINC`。（对应文档「4.5
> 任意修改 PC 指针」的写法）

#### 阶段二：ST0 = 1，正式执行指令

这时控制器看 `IR7_IR4`，决定当前是什么指令。

例如：

```text
IR7_IR4 = 0001 -> ADD
IR7_IR4 = 0101 -> LD
IR7_IR4 = 0111 -> JC
IR7_IR4 = 1110 -> STP
```

然后输出对应控制信号。

---

## 8. 几条典型指令如何执行

下面用几条代表性指令说明控制器是怎么工作的。

---

### 8.1 NOP：空操作

NOP 什么都不算，只取下一条指令。

控制信号：

```text
SHORT = 1
PCINC = 1
LIR = 1
```

含义：

- 本条指令一拍完成；
- PC 加 1；
- 下一条指令装入 IR。

---

### 8.2 ADD：加法

ADD 的核心动作是：

```text
Rd ← Rd + Rs
```

控制信号主要包括：

```text
ABUS = 1     ALU 结果送总线
DRW = 1      写回寄存器
LDZ = 1      更新零标志
LDC = 1      更新进位标志
S = 1001     选择 ALU 加法功能
CIN = 1      配合 TEC-8 ALU 的加法控制
SHORT = 1
PCINC = 1
LIR = 1
```

这里可以看到“流水”的特点：

- 前半部分在执行 ADD；
- 后半部分 `PCINC + LIR` 已经在取下一条指令。

### 为什么 ADD 要更新 Z/C

因为后面的条件跳转 `JZ`、`JC` 要根据标志位判断。

例如：

- 结果为 0，则 `Z=1`；
- 产生进位，则 `C=1`。

所以 ADD 这类算术指令需要更新标志。

---

### 8.3 LD：从存储器读数

LD 的功能是：

```text
Rd ← [Rs]
```

也就是把 `Rs` 中保存的地址拿来访问内存，再把内存数据写进 `Rd`。

它需要两个节拍。

| 节拍 | 控制信号 | 含义 |
| --- | --- | --- |
| W1 | `ABUS + LAR + M + S=1010` | 把地址送入 AR |
| W2 | `MBUS + DRW + PCINC + LIR` | 从内存读数据，写回寄存器，并取下一条指令 |

### 为什么 LD 需要两拍

因为它要先“算出/送出地址”，再“根据地址读内存”。

这两个动作不能完全挤在同一拍里，否则 AR 还没稳定，内存读出的数据就不可靠。

所以 LD 是长周期指令。

---

### 8.4 ST：写存储器

ST 的功能是：

```text
[地址寄存器] ← 源寄存器数据
```

它也需要两个节拍。

| 节拍 | 控制信号 | 含义 |
| --- | --- | --- |
| W1 | `ABUS + LAR + M + S=1111` | 先把目标地址装入 AR |
| W2 | `ABUS + MEMW + M + S=1010 + PCINC + LIR` | 把寄存器数据写入内存，并取下一条指令 |

### 为什么 ST 需要两拍

和 LD 类似，ST 也必须先确定地址，再写数据。

---

### 8.5 JC / JZ：条件跳转

条件跳转需要看标志位。

| 指令 | 条件 | 满足时 |
| --- | --- | --- |
| JC | `C=1` | 执行跳转 |
| JZ | `Z=1` | 执行跳转 |

以 JC 为例：

#### 如果 C=0，不跳转

```text
SHORT = 1
PCINC = 1
LIR = 1
```

含义：当普通短指令处理，直接取下一条。

#### 如果 C=1，跳转

| 节拍 | 控制信号 | 含义 |
| --- | --- | --- |
| W1 | `PCADD` | 根据 offset 修改 PC |
| W2 | `PCINC + LIR` | 跳转后取下一条指令 |

### 为什么条件跳转分两种情况

因为不跳转时非常简单，只要顺序执行下一条即可。

但跳转时需要修改 PC，PC 修改后还要再取新地址处的指令，所以需要更长的过程。

---

### 8.6 JMP：无条件跳转

JMP 不看 C/Z，直接跳。

| 节拍 | 控制信号 | 含义 |
| --- | --- | --- |
| W1 | `LPC + ABUS + M + S=1111` | 把目标地址装入 PC |
| W2 | `PCINC + LIR` | 从跳转目标处取指令 |

---

### 8.7 STP：停机

STP 只需要：

```text
STOP = 1
```

它不再让 PC 前进，也不再装入下一条指令。

---

## 9. 关键控制信号通俗解释

| 信号 | 通俗理解 | 作用 |
| --- | --- | --- |
| `SBUS` | 开关上车 | 数据开关 SD7~SD0 送系统总线 |
| `MBUS` | 内存上车 | 存储器输出送系统总线 |
| `ABUS` | ALU 上车 | ALU 输出送系统总线 |
| `DRW` | 写寄存器 | 把总线数据写入通用寄存器 |
| `MEMW` | 写内存 | 把总线数据写入存储器 |
| `LAR` | 装 AR | 把总线数据装入地址寄存器 |
| `LPC` | 装 PC | 把总线数据装入程序计数器 |
| `LIR` | 装 IR | 把存储器中的指令装入指令寄存器 |
| `PCINC` | PC 加 1 | 指向下一条指令 |
| `ARINC` | AR 加 1 | 指向下一个内存单元 |
| `PCADD` | PC 加偏移 | 条件跳转使用 |
| `LDZ` | 更新 Z | 保存“结果是否为 0” |
| `LDC` | 更新 C | 保存“是否进位/借位” |
| `STOP` | 暂停 | 停机或手动模式下暂停 |
| `SHORT` | 短周期 | 告诉节拍逻辑本指令一拍完成 |
| `M` | ALU 模式 | 区分逻辑/算术操作 |
| `S` | ALU 功能码 | 选择 ALU 具体做什么 |
| `SEL` | 选择寄存器 | 选择读/写哪个寄存器 |
| `SELCTL` | 面板选择控制 | 手动读写寄存器时使用 |

一个重要原则：

> 同一时间通常只能有一个部件主动把数据送上总线，例如 SBUS、MBUS、ABUS 不应随便同时打开。

否则就像几辆车同时抢同一条单车道，可能造成总线冲突。

## 10. 为什么代码使用 `case (1'b1)`

主控制逻辑中有这样的写法：

```verilog
case (1'b1)
    WRITE_REG: begin ... end
    READ_REG:  begin ... end
    READ_MEM:  begin ... end
    WRITE_MEM: begin ... end
    INS_FETCH: begin ... end
endcase
```

这是一种常见写法，意思是：

> 哪个条件为 1，就进入哪个分支。

因为 `WRITE_REG`、`READ_REG` 等模式信号理论上同一时间只有一个为 1，所以这样写可以让模式分支很清楚。

---

## 11. 为什么要区分 W1 和 W2

TEC-8 的一个机器周期中有不同节拍。项目主要使用：

```text
W1
W2
```

很多简单指令只需要 W1：

```text
ADD、SUB、AND、INC、NOP、MOV、CMP、DEC ...
```

一些复杂指令需要 W1 + W2：

```text
LD、ST、JC 跳转、JZ 跳转、JMP
```

### 为什么不全部都做成一拍

因为有些动作有先后依赖。

例如 LD：

1. W1：先把地址装入 AR；
2. W2：再从该地址读内存。

如果硬塞成一拍，地址和数据可能来不及稳定。

### 为什么不全部都做成两拍

当然可以，但会慢。

ADD、SUB 这种寄存器-寄存器操作一拍就能完成，没有必要浪费第二拍。

所以设计成：

- 简单指令：短周期，一拍完成；
- 访存/跳转指令：长周期，两拍完成。

---

## 12. 设计中的几个重要取舍

### 12.1 模式开关先锁存，而不是直接使用

**目的**：稳定。

如果直接使用手拨开关，切换瞬间可能产生错误模式。

代码通过 `Q` 锁存模式，避免开关抖动直接影响控制输出。

---

### 12.2 用 ST0 保存阶段，而不是只靠 W1/W2

**目的**：记住“这是第几轮操作”。

例如读存储器时，每次都是 W1，但第一次 W1 是装地址，后面的 W1 是读数据。

只看 W1 不够，所以需要 ST0。

---

### 12.3 所有控制信号默认清零

**目的**：安全。

只有明确需要的信号才拉高，避免误写寄存器、误写内存、总线冲突。

---

### 12.4 短指令执行时同时取下一条指令

**目的**：提高执行效率。

这就是项目名中“流水”的主要体现。

---

### 12.5 条件跳转不满足时走短周期

**目的**：节省时间。

如果条件不满足，就不需要修改 PC，直接顺序执行下一条即可。

---

## 13. 已知问题和注意事项

项目中已有 `待确认问题.md` 记录了几个注意点。小白阅读时重点知道下面两个就够了。

---

### 13.1 取指阶段 ST0=0 采用单拍短周期写法（原 W2 时序隐患已修复）

当前代码中，取指模式下 `ST0=0` 只有一个 W1，且拉了 `SHORT`，把本机器周期
压成单拍：

```text
ST0=0, W1:
    SBUS + LPC + SST0 + STOP + SHORT
```

这样设计的原因：`SST0` 在取指模式用 W1 触发，`ST0` 在 `T3` 下降沿更新。
早期实现在 `ST0=0` 段同时安排了 W1（装 PC）和 W2（取第一条指令），会导致
W1→W2 切换时 `ST0` 已翻 1，W2 落到执行分支按 `IR7_IR4` 译码，与期望的
`LIR+PCINC` 冲突。改成单拍短周期后，不再有 W2，冲突从根上消除。

那第一条指令什么时候取？

本拍不取指（无 `LIR`）。复位后 IR 通常是 `00H`，而 `00H` 被定义为 NOP。
下一周期 `ST0=1` 时，自然走 NOP 分支，输出 `LIR + PCINC + SHORT`，由此
完成初始进入程序。这与文档「4.5 任意修改 PC 指针」的思路一致：
利用 NOP 指令完成初始进入程序操作。

---

### 13.2 NOT 指令的 ALU 功能码待确认

代码中 `NOT` 指令设置了：

```text
M = 1
```

但没有明确设置 `S`，所以 `S` 保持默认 `0000`。

根据已有分析，TEC-8 中 `M=1, S=0000` 很可能不是“取反”，而是“传送 A”。

因此当前 `NOT` 可能并没有真正完成按位取反，需要查完整 TEC-8 ALU 功能表后再修正。

## 14. 初学者如何从代码对应到硬件现象

建议按下面顺序理解。

### 第一步：先看模式

先判断当前 `SWC_SWB_SWA` 是什么。

例如：

```text
100 -> 写寄存器
001 -> 写存储器
000 -> 执行程序
```

### 第二步：再看 ST0

判断当前是第一阶段还是第二阶段。

例如写存储器：

```text
ST0=0 -> 当前数据开关表示地址
ST0=1 -> 当前数据开关表示要写入的数据
```

### 第三步：再看 W1/W2

判断当前节拍做哪一步。

例如 LD：

```text
W1 -> 送地址到 AR
W2 -> 从内存读数据到寄存器
```

### 第四步：最后看具体控制信号

例如看到：

```text
SBUS = 1
LAR = 1
```

就理解为：

```text
数据开关上的值经过总线装入 AR
```

看到：

```text
ABUS = 1
DRW = 1
```

就理解为：

```text
ALU 的结果经过总线写回寄存器
```

看到：

```text
MBUS = 1
DRW = 1
```

就理解为：

```text
内存读出的数据经过总线写入寄存器
```

## 15. 一个完整例子：执行 ADD

假设当前 IR 中是：

```text
ADD R0, R1
```

也就是：

```text
R0 ← R0 + R1
```

控制器在 W1 输出：

```text
ABUS = 1
DRW = 1
LDZ = 1
LDC = 1
S = 1001
CIN = 1
SHORT = 1
PCINC = 1
LIR = 1
```

可以一步步翻译：

1. `S=1001, CIN=1`：告诉 ALU 做加法；
2. `ABUS=1`：ALU 结果送到系统总线；
3. `DRW=1`：把总线结果写回目标寄存器；
4. `LDZ=1, LDC=1`：保存零标志和进位标志；
5. `PCINC=1`：PC 指向下一条指令；
6. `LIR=1`：下一条指令进入 IR；
7. `SHORT=1`：这条指令一拍完成。

所以 ADD 一拍里完成了：

```text
计算 + 写回 + 更新标志 + 取下一条指令
```

这就是这个控制器最核心的工作方式。

## 16. 总结

这个项目实现了一个可以下载到 TEC-8 CPLD 的流水硬布线控制器。

它完成的事情包括：

1. 支持手动写寄存器、读寄存器、写存储器、读存储器；
2. 支持从指定地址开始取指并执行程序；
3. 根据 `IR7_IR4` 译码执行多种指令；
4. 对简单指令使用短周期，一拍完成；
5. 对访存和跳转指令使用两拍完成；
6. 在执行指令的同时取下一条指令，体现简单流水思想；
7. 通过 `Q` 锁存模式，通过 `ST0` 保存阶段，使控制流程稳定可控；
8. 使用硬布线逻辑直接生成控制信号，适合 TEC-8 和 EPM7128 CPLD。

如果只记住一句话，可以记这个：

> 这个 Verilog 模块就是 TEC-8 的“指挥员”：它根据模式、节拍、指令和标志位，在每一拍输出正确的控制信号，让数据在寄存器、ALU、内存、PC、IR 之间按实验流程移动。
