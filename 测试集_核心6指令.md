# 核心 6 指令测试集（仅用 MOV / ADD / JC / LD / ST / STOP）

本测试用一个短程序，**只从以下 6 种指令中选取，可重复使用，程序中不出现集合外指令**：

| # | 指令 | 本集用途 |
| - | ---- | -------- |
| 1 | MOV  | 建立地址指针；JC 陷阱改写 R3 |
| 2 | ADD  | 运算写回；溢出造 `C=1` |
| 3 | JC   | 条件跳转（依赖 ADD 的 C） |
| 4 | LD   | 读存储器 |
| 5 | ST   | 写存储器 |
| 6 | STOP | 停机 |

不覆盖集合外指令：NOP、SUB、AND、INC、JZ、JMP、CMP、NOT、DEC 等均**不得出现在本程序机器码中**。

定位：比 `测试集.md`（15 条全覆盖）更短，适合上板冒烟、排查取指/写回/跳转/访存主路径。

控制器源码见 `pipelined_hardwired_controller.v`，指令定义见 `指令格式.md`，执行流程见
`流程图.md`，74181 功能表见 `74ls181.md`。

> 操作前必读：`测试操作注意事项.md`（陷阱 #1「模式锁存两步法」、陷阱 #2「用灯组合判断
> ST0 阶段」）与 `TEC-8寄存器读写时序说明.md`。

---

## 0. 设计思路

- **指令集合封闭**：程序区每个字节的 opcode 都属于上述 6 种之一；陷阱填充也用 `MOV`，不用 NOP。
- **ADD 溢出制造 `C=1`**：`0xFF + 0x02 = 0x01`（进位 `C=1`，结果非 0 故 `Z=0`），紧接 `JC`。
- **JC 跳过的区域设陷阱（仍用 MOV）**：
  - 0x03 `MOV R3,R1`：JC 失效时 `R3←0x02`（正常 `R3=0x20`）
  - 0x04 `MOV R3,R1`：第二道填充，同样只在 JC 失效时执行；与 0x03 同为 MOV，不引入新指令
- **LD 读已知数据**：`MEM[0x20]` 预置 `0xAA`，`LD R0,[R2]` 后 `R0=0xAA`。
- **ST 覆盖同一地址**：`ST R1,[R2]` 把 `R1=0x02` 写入 `MEM[0x20]`。
- **无 JMP**：顺序落到 STOP，只验证条件跳转 JC。
- **MOV 复用 3 次**：主路径 1 次（建指针）+ 陷阱区 2 次（失败诊断），体现「可重复使用」。

## 1. 寄存器初值（写寄存器模式 SWC_SWB_SWA=100）

| 寄存器 | 初值     | 用途                                   |
| ------ | -------- | -------------------------------------- |
| R0     | 0xFF     | ADD 目的操作数（0xFF+0x02 溢出造 C=1） |
| R1     | 0x02     | ADD 源操作数；ST 写入的数据；陷阱源    |
| R2     | 任意→0x00| 将被 `MOV R2,R3` 覆盖为 0x20           |
| R3     | 0x20     | LD/ST 数据区地址；MOV 源；陷阱目标     |

## 2. 内存映像（写存储器模式 SWC_SWB_SWA=001）

### 2.1 程序区（0x00 起，连续 8 字节，仅含 6 种指令）

| 地址  | 机器码 | 指令        | 所属集合 | IR7..IR0 拆解 | 说明 |
| ----- | ------ | ----------- | -------- | ------------- | ---- |
| 0x00  | 0xCB   | MOV R2, R3  | MOV      | 1100 10 11    | 主路径：R2←0x20 |
| 0x01  | 0x11   | ADD R0, R1  | ADD      | 0001 00 01    | 主路径：造 C=1 |
| 0x02  | 0x72   | JC +2       | JC       | 0111 0010     | 主路径：跳到 0x05 |
| 0x03  | 0xCD   | MOV R3, R1  | MOV      | 1100 11 01    | 陷阱：JC 失效才执行 |
| 0x04  | 0xCD   | MOV R3, R1  | MOV      | 1100 11 01    | 填充：仍属 MOV，不引入新指令 |
| 0x05  | 0x52   | LD R0, [R2] | LD       | 0101 00 10    | 主路径：读 MEM[0x20] |
| 0x06  | 0x69   | ST R1, [R2] | ST       | 0110 10 01    | 主路径：写 MEM[0x20] |
| 0x07  | 0xE0   | STOP        | STOP     | 1110 00 00    | 停机 |

**指令种类统计（程序区）**：MOV×3、ADD×1、JC×1、LD×1、ST×1、STOP×1 → 共 6 种，无集合外指令。

### 2.2 数据区（0x20，LD 读取 + ST 写入）

| 地址 | 初值 | 说明                          |
| ---- | ---- | ----------------------------- |
| 0x20 | 0xAA | LD 读出特征值，与寄存器初值均不同 |

> ST 执行后 `MEM[0x20]` 被改写为 `R1=0x02`，见第 5 节读存储器验证。

## 3. 控制流与跳转计算

`@` = 当前指令 PC + 1；4 位有符号 offset（IR3 为符号位，0 正 1 负）。

| 跳转指令 | 当前 PC | @    | offset | 目标 = @ + offset | 效果            |
| -------- | ------- | ---- | ------ | ----------------- | --------------- |
| JC +2    | 0x02    | 0x03 | +2     | 0x05              | 跳过 0x03、0x04 |

执行路径：
```
0x00 MOV → 01 ADD → 02 JC ──C=1──▶ 05 LD → 06 ST → 07 STOP
                      │
           (0x03/04 MOV R3,R1 陷阱被跳过)
```

跳转失效诊断：

| 实测 R3 | 判定 |
| ------- | ---- |
| 0x20    | JC 正常 ✓ |
| 0x02    | JC 未跳转（误执行了 0x03/0x04 `MOV R3,R1`） |

---

## 4. 执行流程

| 步 | PC  | 机器码 | 指令       | 操作           | R0  | R1  | R2  | R3  | Z   | C   | 说明 |
| -- | --- | ------ | ---------- | -------------- | --- | --- | --- | --- | --- | --- | ---- |
| 初 | —   | —      | (初值)     | 写寄存器装入   | FF  | 02  | 00  | 20  | x   | x   | 标志初值任意 |
| 1  | 0x00| 0xCB   | MOV R2,R3  | R2 ← R3        | FF  | 02  | 20  | 20  | x   | x   | R2=0x20 |
| 2  | 0x01| 0x11   | ADD R0,R1  | R0 ← R0+R1     | 01  | 02  | 20  | 20  | 0   | 1   | 0xFF+0x02→0x01，**造 C=1** |
| 3  | 0x02| 0x72   | JC +2      | C=1 → PC=0x05  | 01  | 02  | 20  | 20  | 0   | 1   | **JC 成立**，跳过 0x03/04 |
| 4  | 0x05| 0x52   | LD R0,[R2] | R0 ← MEM[0x20] | AA  | 02  | 20  | 20  | 0   | 1   | 读出 0xAA |
| 5  | 0x06| 0x69   | ST R1,[R2] | MEM[0x20] ← R1 | AA  | 02  | 20  | 20  | 0   | 1   | 写入 0x02 |
| 6  | 0x07| 0xE0   | STOP       | 停机           | AA  | 02  | 20  | 20  | 0   | 1   | 停机 |

## 5. 最终预期结果

### 5.1 寄存器与标志

| 项目 | 预期值 | 来源 |
| ---- | ------ | ---- |
| R0   | 0xAA   | LD 从 MEM[0x20] 读入 |
| R1   | 0x02   | 程序运行中未改写 |
| R2   | 0x20   | 0x00 处 MOV R2,R3 |
| R3   | 0x20   | 初值；JC 正常时未改写 |
| Z    | 0      | 最后更新标志的是 ADD，结果 0x01≠0 |
| C    | 1      | ADD 产生进位 |

> LD/ST/MOV/JC/STOP 不更新 Z/C，停机时标志仍由 ADD 决定。

### 5.2 内存

| 地址 | 最终值 | 说明 |
| ---- | ------ | ---- |
| 0x20 | 0x02   | ST 写入（原 0xAA 被覆盖） |
| 其他 | 不变   | 程序区保持写入的机器码 |

## 6. 通过判据与异常对照

### 6.1 通过判据

- R0=0xAA，R1=0x02，R2=0x20，R3=0x20
- MEM[0x20]=0x02
- Z=0，C=1

全部符合 → 本集 6 种指令功能通过。

### 6.2 常见异常对照

| 现象 | 可能原因 |
| ---- | -------- |
| R3=0x02 | JC 未跳转：核对 ADD 是否造出 C=1（R0 初值 0xFF、R1=0x02）、0x02 处编码 `0x72` |
| R0 终值仍为 0x01（非 0xAA） | LD 未执行或地址错：核对 R2=0x20、MEM[0x20] 初值 0xAA、0x05 处 `0x52` |
| R0=0xAA 但 MEM[0x20] 仍为 0xAA | ST 未写成功：核对 0x06 处 `0x69`、R1 是否仍为 0x02 |
| R2≠0x20 | MOV 失败：核对 0x00 处 `0xCB`、R3 初值 0x20 |
| 程序未停机 / PC 跑飞 | 机器码写偏、PC 起始未装 0x00、或模式切换未用两步法 |

### 6.3 封闭性自检

装入程序后，确认每个程序字节的高 4 位仅属于：

| IR7..IR4 | 指令 |
| -------- | ---- |
| 1100     | MOV  |
| 0001     | ADD  |
| 0111     | JC   |
| 0101     | LD   |
| 0110     | ST   |
| 1110     | STOP |

若出现其它高 4 位（如 0000=NOP、0010=SUB、1000=JZ、1001=JMP 等），则违反本集约束。

## 7. 上板操作要点（摘要）

1. 写寄存器：按「①a 锁存模式 → ①b 起写数据」两步法，装入 R0=0xFF、R1=0x02、R3=0x20（R2 任意）。
2. 写存储器：两步法锁存 001 后，从 AR=0x00 起连续写入程序 8 字节；再把 AR 拨到 0x20 写入数据 `0xAA`。
3. 取指模式：锁存 000，LPC 装 PC=0x00，连续运行或单步到 STOP。
4. 读寄存器核对 R0~R3；读存储器核对 MEM[0x20]=0x02。

详细灯判据与时序见 `测试操作注意事项.md`、`TEC-8寄存器读写时序说明.md`。
